## ❗ ที่ยังไม่ได้ตัดสินใจ

### 1. ประเภทงาน — AI เดาไปก่อน

ตอนนี้ใส่ไว้ 4 แบบ (พี่ดิวยังไม่ได้ยืนยัน):

| ค่าใน DB | ภาษาไทย |
|---|---|
| `hourly` | รายชั่วโมง |
| `daily` | รายวัน (ไป-กลับ) |
| `overnight` | ค้างคืน |
| `live_in` | อยู่ประจำ |

> ถ้าไม่ตรงกับที่คิดไว้ แก้ที่ `db/schema.sql` (ENUM) + `public/js/app.js` (`CARE_TYPE_TH` และ dropdown)

### 2. โมเดลรายได้ — ยังไม่คิด

MVP ไม่เก็บเงินเลย ซึ่งถูกแล้วสำหรับการเอาไปโชว์
แต่พออนาคตจะเก็บเงิน จะเจอปัญหาคลาสสิกของแอพจับคู่ทุกตัว:

> **"คู่ค้าหนีออกนอกแอพ"** — เราให้เขาแชทกัน แลกเบอร์ แล้วจ่ายเงินกันเอง
> ครั้งแรกใช้แอพ ครั้งต่อไปโทรหากันตรง ๆ **แอพไม่ได้อะไรเลย**

**ทางเลือก:**

| วิธี | ข้อดี | ข้อเสีย |
|---|---|---|
| **ระบบตั๋ว** (แบบ ChaungThai) | เก็บตอน "กดขอรับงาน" — เก็บได้ก่อนที่เขาจะหนี | แคร์กิฟเวอร์ต้องจ่ายก่อนได้งาน อาจไม่ชอบ |
| **Escrow / หัก %** | ได้ % จากทุกงาน สมเหตุสมผล | ต้องมี payment gateway + KYC ทางการเงิน + เรื่องกฎหมาย |
| **Subscription** | รายได้สม่ำเสมอ | ต้องมีคุณค่าพอให้จ่ายรายเดือน |
| **ทำให้อยู่ในแอพคุ้มกว่า** | ประกันอุบัติเหตุ · เช็คอิน-เช็คเอาท์ · ปุ่มฉุกเฉิน · ประวัติการทำงาน | ต้องลงทุนสร้างฟีเจอร์ |

> 💡 **DB ออกแบบเผื่อไว้แล้ว** — เพิ่มตาราง `tickets` / `payments` ทีหลังได้โดยไม่ต้องรื้อของเดิม

## 🔒 ต้องทำก่อนเปิดใช้จริง (ไม่ใช่แค่เดโม)

| เรื่อง | ความเสี่ยงถ้าไม่ทำ |
|---|---|
| **Rate limit ที่ login** | โดนยิงเดารหัสผ่านรัว ๆ |
| **CSRF token** | เว็บอื่นหลอกให้ browser ยิง API แทนผู้ใช้ได้ |
| **SSL ของจริง** (Let's Encrypt) | ตอนนี้ self-signed เบราว์เซอร์ขึ้นเตือน |
| **ตรวจ KYC ให้จริงจัง** | ตอนนี้แอดมิน "ดูรูปด้วยตา" — ของจริงควรต่อ API ตรวจบัตร ปชช. + ตรวจประวัติอาชญากรรม |
| **PDPA** | เก็บบัตรประชาชน + ที่อยู่ + พิกัด = ข้อมูลส่วนบุคคลอ่อนไหว ต้องมีนโยบายความเป็นส่วนตัว + ขอความยินยอม |
| **สำรองข้อมูลอัตโนมัติ** | ข้อมูลหายแล้วหายเลย |
| **ลบรูป KYC เมื่อไม่จำเป็น** | เก็บบัตรประชาชนไว้ตลอดกาล = ความเสี่ยงสูง ถ้าโดนแฮก |

## 📱 Phase 3 — Flutter App

เมื่อโชว์ผ่าน และตัดสินใจทำของจริงแล้ว

**ข่าวดี:** Backend ที่ทำไว้ **ใช้ต่อได้เลยทั้งหมด** — Flutter แค่เรียก API ชุดเดิม
สิ่งที่ต้องเปลี่ยน:
- Auth จาก cookie → **Bearer token** (มือถือใช้ cookie ไม่สะดวก)
- เพิ่ม Firebase Cloud Messaging สำหรับ push
- แผนที่ใช้ `flutter_map` (Leaflet เวอร์ชัน Flutter — ใช้ OSM เหมือนกัน ฟรีเหมือนกัน)
