## ❗ ที่ยังไม่ได้ตัดสินใจ

### 1. ประเภทงาน — AI เดาไปก่อน ยังไม่เคยยืนยัน

| ค่าใน DB | ภาษาไทย |
|---|---|
| `hourly` | รายชั่วโมง |
| `daily` | รายวัน (ไป-กลับ) |
| `overnight` | ค้างคืน |
| `live_in` | อยู่ประจำ |

> ถ้าไม่ตรงกับที่คิดไว้ ต้องแก้ 3 ที่: ENUM ใน DB (เขียน `004_xxx.sql` ใหม่) ·
> `CARE_TYPES` ใน `src/routes/{jobs,hires}.js` · `CARE_TYPE_TH` + dropdown ใน `public/js/`

### 2. โปรไฟล์ 31 ช่อง — ใส่เยอะไว้ก่อนตามที่สั่ง รอตัดออก

ตอนนี้มี: ส่วนตัว (10) · บัตร ปชช. (3) · ติดต่อ (4) · ที่อยู่ตามบัตร (5) · ที่อยู่ปัจจุบัน (6) · ผู้ติดต่อฉุกเฉิน (3) · อื่น ๆ (2)
**ทุกช่อง NULL ได้** → ตัดออกได้ทุกเมื่อโดยไม่พังของเดิม

### 3. โมเดลรายได้ — ยังไม่คิด

MVP ไม่เก็บเงินเลย ซึ่งถูกแล้วสำหรับการเอาไปโชว์
แต่พออนาคตจะเก็บเงิน จะเจอปัญหาคลาสสิกของแอพจับคู่ทุกตัว:

> **"คู่ค้าหนีออกนอกแอพ"** — เราให้เขาแชทกัน แลกเบอร์ แล้วจ่ายเงินกันเอง
> ครั้งแรกใช้แอพ ครั้งต่อไปโทรหากันตรง ๆ **แอพไม่ได้อะไรเลย**

| วิธี | ข้อดี | ข้อเสีย |
|---|---|---|
| **ระบบตั๋ว** (แบบ ChaungThai) | เก็บตอน "กดขอรับงาน" — เก็บได้ก่อนที่เขาจะหนี | แคร์กิฟเวอร์ต้องจ่ายก่อนได้งาน อาจไม่ชอบ |
| **Escrow / หัก %** | ได้ % จากทุกงาน สมเหตุสมผล | ต้องมี payment gateway + KYC ทางการเงิน + กฎหมาย |
| **Subscription** | รายได้สม่ำเสมอ | ต้องมีคุณค่าพอให้จ่ายรายเดือน |
| **ทำให้อยู่ในแอพคุ้มกว่า** | ประกันอุบัติเหตุ · เช็คอิน-เช็คเอาท์ · ปุ่มฉุกเฉิน · ประวัติการทำงาน | ต้องลงทุนสร้างฟีเจอร์ |

> 💡 **DB เผื่อไว้แล้ว** — เพิ่มตาราง `tickets` / `payments` ทีหลังได้โดยไม่ต้องรื้อของเดิม

## 🔒 ต้องทำก่อนเปิดใช้จริง (ไม่ใช่แค่เดโม)

| เรื่อง | ความเสี่ยงถ้าไม่ทำ | สถานะ |
|---|---|---|
| **เปลี่ยนรหัส SSH (`server_live` + `root`)** | รหัสหลุดในแชท AI แล้ว | 🔴 **ค้าง** |
| **ทำ repo เป็น private** หรือย้ายเอกสารออก | IP + ผังเครื่อง + รายการช่องโหว่ อยู่บน GitHub สาธารณะ | 🔴 **ค้าง** |
| **เปิด KYC ของจริง** | ตอนนี้ใครกดปุ่มก็ผ่าน = ใครก็เห็นที่อยู่บ้านคนแก่ | 🔴 **ค้าง** |
| **Rate limit ที่ login** | โดนยิงเดารหัสผ่านรัว ๆ | 🔴 |
| **CSRF token** | เว็บอื่นหลอกให้ browser ยิง API แทนผู้ใช้ได้ | 🔴 |
| **SSL ของจริง** (Let's Encrypt) | ตอนนี้ self-signed เบราว์เซอร์ขึ้นเตือน | 🔴 |
| **PDPA** | เก็บบัตร ปชช. + ที่อยู่ + พิกัด = ข้อมูลอ่อนไหว ต้องมีนโยบายความเป็นส่วนตัว + ขอความยินยอม | 🔴 |
| **ลบข้อมูลบัตร ปชช. เมื่อไม่จำเป็น** | เก็บไว้ตลอดกาล = ความเสี่ยงสูงถ้าโดนแฮก | 🔴 |
| **สำรองข้อมูลอัตโนมัติ** | ข้อมูลหายแล้วหายเลย | 🔴 |
| **ตรวจ KYC ให้จริงจัง** | ของจริงควรต่อ API ตรวจบัตร ปชช. + ตรวจประวัติอาชญากรรม | 🔴 |

> ### 💡 อันที่ควรทำก่อนเพื่อน
> **เปิด KYC ของจริงกลับมา** — เพราะทั้งแอพนี้ขายอยู่บนประโยคเดียวคือ
> *"คนที่ยังไม่ผ่านการตรวจสอบ จะไม่เห็นที่อยู่บ้านคนแก่"*
> ถ้าใครกดปุ่มก็ผ่านได้ ประโยคนั้นก็ไม่จริง — เดโมพอไหว แต่เปิดใช้จริงไม่ได้
